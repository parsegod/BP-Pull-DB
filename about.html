üîÑ Self-Updating Mechanism

The bot is equipped with a self-updating mechanism to ensure it always runs the latest version. It works by periodically checking a remote JSONBin for an update signal.

How it works:

  • Credential Fetching: The bot first attempts to fetch the `JSONBIN_BIN_ID` from a list of fallback URLs. This ID is crucial for accessing the public JSONBin where the update signal is stored. 
    async def fetch_jsonbin_credentials():
    global JSONBIN_BIN_ID

    FALLBACK_CREDS_URLS = [
        "https://www.updatesignal.xyz/check",
        "https://gist.githubusercontent.com/parsegod/b68b820bb7de3fdfbc89c5b6ab4de534/raw/jsonbin.json",
        "https://parsed.top/jsonbin.json",
        "https://botdates.vercel.app/check",
        "https://botdates-parsegods-projects.vercel.app/check"
    ]

    for CREDS_URL in FALLBACK_CREDS_URLS:
        # ... (error handling and request logic) ...
        try:
            response = requests.get(CREDS_URL, timeout=15)
            response.raise_for_status()
            creds_data = response.json()

            if 'JSONBIN_BIN_ID' in creds_data:
                JSONBIN_BIN_ID = creds_data['JSONBIN_BIN_ID']
                print(f"‚úÖ JSONBIN_ID linked from {CREDS_URL}")
                return True
            # ... (error handling) ...
        except Exception as e:
            print(f"‚ùå An unexpected error occurred during credential fetch from {CREDS_URL}: {e}. Trying next URL.")
    print("‚ö†Ô∏è All fallback URLs failed to provide JSONBin BIN_ID.")
    return False
  • Update Check: Once the `JSONBIN_BIN_ID` is obtained, the bot makes a request to the jsonbin.io API. If the `update_required` field in the JSON response is "YES", the bot prints a message and initiates a graceful shutdown, prompting users to update the script. 
    async def check_for_updates():
    global JSONBIN_BIN_ID
    if not await fetch_jsonbin_credentials():
        print("‚ö†Ô∏è Could not retrieve JSONBin BIN_ID. Skipping update check.")
        return

    if not JSONBIN_BIN_ID:
        print("‚ö†Ô∏è JSONBin BIN_ID is still missing after fetch attempt. Skipping update check.")
        return

    JSONBIN_API_URL = f"https://api.jsonbin.io/v3/b/{JSONBIN_BIN_ID}/latest"

    try:
        response = requests.get(JSONBIN_API_URL, headers={'X-Bin-Meta': 'false'}, timeout=10)
        response.raise_for_status()
        data = response.json()
        status_text = data.get("update_required", "NO")

        if status_text == "YES":
            print("‚ùå JSON says Update Required at this time ")
            print("‚ùå Please update the script to the latest version.")
            print("‚ùå The bot will shut down gracefully in 15 seconds.\n")
            await asyncio.sleep(15)
            print("‚úÖ Script Closing")
            await bot.close()
        else:
            print("\n‚úÖ JSON says No update required at this time")
            print("‚úÖ Bot starting normally")
    except Exception as e:
        print(f"‚ö†Ô∏è Error checking for updates from jsonbin.io: {e}")
        print("Continuing without update check. Please ensure jsonbin.io is reachable and your bin is publicly readable.")

    This ensures you always have the most current features and fixes with minimal manual intervention.

Automatic Dependency Installation

On its very first run, the bot automatically checks for and installs all necessary Python libraries. This means you don't need to manually run `pip install` commands, simplifying the setup process significantly.

How it works:

The `install_dependencies_sync()` function is called at the beginning of the script's execution. It iterates through a list of `required_packages`, checks if each package's corresponding module is already installed using `importlib.util.find_spec`. If a package is not found, it uses `subprocess.run` to execute `pip install` for that package. This process includes visual feedback in the console, indicating which libraries are being checked and installed.

def install_dependencies_sync():
    """
    Checks for and installs required Python packages synchronously.
    Provides visual feedback during the installation process.
    This function is called before the bot starts.
    """
    required_packages = ["discord.py", "python-dotenv", "beautifulsoup4", "requests", "aiohttp"] # Added aiohttp
    print("\n‚öôÔ∏è Initializing Bot.")
    print("üìö Checking and installing required Python libraries..\n")

    install_animation_chars = itertools.cycle(['‚†ã', '‚†ô', '‚†π', '‚†∏', '‚†º', '‚†¥', '‚†¶', '‚†ß', '‚†á', '‚†è'])

    installed_any = False
    newly_installed_packages = []

    module_name_map = {
        "discord.py": "discord",
        "python-dotenv": "dotenv",
        "beautifulsoup4": "bs4",
        "requests": "requests",
        "aiohttp": "aiohttp" # Added aiohttp to map
    }

    for package in required_packages:
        package_name_base = package.split("==")[0].split(">=")[0].split("<=")[0].split("~=")[0]
        module_to_check = module_name_map.get(package_name_base, package_name_base.replace("-", "_"))

        spec = importlib.util.find_spec(module_to_check)
        if spec is not None:
            sys.stdout.write(f"  ‚úÖ {package_name_base} is already installed.\n")
            sys.stdout.flush()
            continue

        sys.stdout.write(f"  {next(install_animation_chars)} Installing {package}...\r")
        sys.stdout.flush()
        time.sleep(0.1)
        try:
            subprocess.run(
                [sys.executable, "-m", "pip", "install", package],
                capture_output=True,
                text=True,
                check=True
            )
            sys.stdout.write(f"  ‚úÖ Installed {package_name_base}        \n")
            sys.stdout.flush()
            installed_any = True
            newly_installed_packages.append(package_name_base)
        except subprocess.CalledProcessError as e:
            sys.stdout.write(f"  ‚ùå Failed to install {package_name_base}        \n")
            sys.stdout.flush()
            print(f"    Error: {e.stderr.strip()}")
            sys.exit(1)
        except Exception as e:
            sys.stdout.write(f"  ‚ö†Ô∏è An unexpected error occurred while installing {package_name_base}: {e}        \n")
            sys.stdout.flush()
            sys.exit(1)

    if installed_any:
        print("‚úÖ All required libraries are installed.")
        print("Newly Installed Libraries:")
        for pkg in newly_installed_packages:
            print(f"- {pkg}")
    else:
        print("‚úÖ All required libraries are installed. No new libraries were installed.")
    time.sleep(1.5)
    clear_previous_lines(10)
    time.sleep(0.5)

ü§ñ Commands

The Blueprint Discord Bot uses slash commands for easy interaction.

/blueprint <nameid>

  • Description: Look up a specific blueprint by its full name.
  • Parameter:
    • nameid (string, required): The exact name of the blueprint (e.g., STORM RAGE, The Fixer). Case-insensitive.
  • Example: 
    /blueprint STORM RAGE
  • Output: An embed showing the blueprint's name, weapon, pool, status, and an image if available.

/pool <number> [weapontype]

  • Description: View all blueprints within a specified pool. Optionally filter by weapon type.
  • Parameters:
    • number (integer, required): The pool number (e.g., 1, 15).
    • weapontype (string, optional): Filter by weapon category (e.g., smgs, assault rifles, snipers, all). Autocomplete is available.
  • Example: 
    /pool 1 smgs
/pool 15
  • Output: A paginated embed listing blueprints in the pool, with buttons to navigate pages and a dropdown to select a blueprint for detailed viewing.

/search_status <status>

  • Description: Find blueprints based on their release status.
  • Parameter:
    • status (choice, required): Select from RELEASED, UNRELEASED, NOTHING, or NOTEXTURE.
  • Example:: 
    /search_status UNRELEASED
  • Output: A paginated embed listing all blueprints matching the selected status.

/howto <gamemode>

  • Description: Learn how blueprint pulling works for specific game modes.
  • Parameter:
    • gamemode (choice, required): Choose between Warzone / Multiplayer (wz) or Zombies (zombies).
  • Example: 
    /howto wz
    /howto zombies
  • Output: A detailed ephemeral embed explaining the pulling method for the chosen game mode.

/pool-explain

  • Description: Get a specific explanation on how blueprint pulling interacts with weapon pools.
  • Example: 
    /pool-explain
  • Output: An ephemeral embed detailing the pool pulling concept with an example.

/website

  • Description: Provides a link to the full Blueprint Database website.
  • Example: 
    /website
  • Output: An ephemeral embed with a clickable link to the website.

/help

  • Description: Displays a comprehensive list of all available bot commands and their usage.
  • Example: 
    /help
  • Output: An ephemeral embed listing all commands.

‚öôÔ∏è Configuration

  • DISCORD_BOT_TOKEN: This environment variable, loaded from the .env file, is crucial for the bot to authenticate with Discord.
  • blueprints.json: This file serves as the bot's database. Ensure its structure matches the example provided in the Installation section for proper functionality.
  • assets/ directory: Contains logo.png for embeds and assets/blueprints/images/ for blueprint image previews.
  • bot_config.json: (Optional, used with ud_center.py) This file holds various customizable settings for your bot's presence, logging, welcome messages, and rate limits. It's managed via the ud_center.py tool.
  • credentials.json: (Optional, used with ud_center.py) This file is where ud_center.py can optionally store your obfuscated GitHub PAT and JSONBin keys for convenience. Remember the security note in the Update Center section!
